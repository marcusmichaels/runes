Bitwise AND

/ˈbɪtwaɪz and/

&

a & b compares each bit position — the result is 1 only where both inputs are 1.

Think of a number as a row of on/off switches. & takes two rows and asks: which switches are on in both? Only those survive. Everything else gets zeroed out.

That sounds abstract until you realise you already use it. Hex colours like #3B82F6 pack three values (red, green, blue) into one number — & is how you pull a single channel back out. File permissions (rwx) cram three yes/no answers into one number — & is how you check one without touching the others. Any time you’d otherwise write a pile of booleans, & lets you ask “is this specific thing set?” in a single operation.

Think of it like a stencil; you lay the mask over the value, and only the bits that line up with the holes come through.

Bit by bit

Each number is an 8-bit binary row. Slide a and b to see their bits light up — the result row keeps only the columns where both are 1. It’s the same idea as a.filter(x => b.includes(x)) but at the bit level.

173 & 214 = 132

Masking

Hex colours pack three channels into one integer. Masking with & 0xFF extracts a single byte — the same way you’d slice a substring, but for bits. Drag the colour or pick a channel to see it pulled apart.

The modulo shortcut

When the divisor is a power of 2, n & (d - 1) gives the same result as n % d. The mask is all 1s in the lower bits — it just keeps the remainder. Ring buffers, hash tables, and colour clamping all lean on this.

Checking flags

Permissions are often packed as bits: READ = 4, WRITE = 2, EXEC = 1. AND with a single flag isolates that bit — if the result is non-zero, the permission is set.

It’s just intersection

In arrays you’d write a.filter(x => b.includes(x)) to keep only shared items. & does the same thing, one bit per “item”. Toggle items in each set to see only the overlap survive.

// Finding what two sets have in common:

// With arrays
const shared = a.filter(x => b.includes(x));

// With bit flags — same concept, packed into integers
const ADMIN = 1, EDITOR = 2, VIEWER = 4, MOD = 8;

// Combine flags with | (bitwise OR — sets bits)
const user = ADMIN | EDITOR | VIEWER; // 7  (0b0111)
const required = ADMIN | MOD;         // 9  (0b1001)

// Check overlap with & (keeps shared bits)
const overlap  = user & required;     // 1  (ADMIN only)

if (overlap) {
  /* user has at least one required role */
}
