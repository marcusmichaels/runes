Bitwise OR

/ˈbɪtwaɪz ɔːr/

|

a | b compares each bit position — the result is 1 where either (or both) inputs are 1.

If AND is intersection, then OR is union. It keeps every bit that’s on in either row. Nothing gets zeroed out — bits only get added.

You’ll reach for it whenever you need to combine things: stacking multiple permission flags into one value, turning on features without touching the rest, or merging two sets of options into one config integer.

It’s the opposite of a stencil — instead of blocking, it lets everything through from both sides.

Bit by bit

Each number is an 8-bit binary row. Slide a and b to see their bits light up — the result row keeps every column where either is 1. It’s the same idea as [...new Set([...a, ...b])] but at the bit level.

173 | 214 = 255

Combining flags

Permissions are often packed as bits: READ = 4, WRITE = 2, EXEC = 1. OR combines individual flags into one value — each checked flag contributes its bit to the result. Toggle them to see bits stack up.

READ (4)
WRITE (2)
EXEC (1)
READ | EXEC = 5

| sets bits. & checks them. Together they’re a complete read/write system for packed flags.

Masks & hex notation

A mask is just a number with specific bits set. You’ll often see them written in hex (0x__) because each hex digit maps cleanly to 4 bits. The pattern is the same in both halves — 1, 2, 4, 8 — it just carries left once you overflow the first digit. 

// Each mask lights up exactly one bit:
0x01 =   1 = 0000 0001  (bit 0)
0x02 =   2 = 0000 0010  (bit 1)
0x04 =   4 = 0000 0100  (bit 2)
0x08 =   8 = 0000 1000  (bit 3)
0x10 =  16 = 0001 0000  (bit 4)
0x20 =  32 = 0010 0000  (bit 5)
0x40 =  64 = 0100 0000  (bit 6)
0x80 = 128 = 1000 0000  (bit 7)

OR with a mask turns ON that bit without affecting the others. Bits that were already on stay on — the mask only adds.

Defaults and fallbacks

A common pattern: options = userOpts | defaults fills in missing flags. Bits the user already has stay set. Bits they’re missing get filled in from defaults.

It’s just union

In arrays you’d write [...new Set([...a, ...b])] to merge both sets. | does the same thing, one bit per “item”.

Click any cell in the A or B rows to toggle it. 

// Finding everything from either set:

// With arrays
const merged = [...new Set([...a, ...b])];

// With bit flags — same concept, packed into integers
const ADMIN = 1, EDITOR = 2, VIEWER = 4, MOD = 8;

// Combine with | (bitwise OR — adds bits together)
const teamA = ADMIN | EDITOR;   // 3  (0b0011)
const teamB = EDITOR | VIEWER;  // 6  (0b0110)

const allRoles = teamA | teamB; // 7  (0b0111) — everyone's roles merged

& (bitwise AND) is how you check flags — | is how you combine them. See Bitwise AND →
